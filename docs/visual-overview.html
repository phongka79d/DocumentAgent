Project Purpose

This milestone creates the baseline project layout and conventions so later implementation agents do not guess file locations or invent incompatible patterns.

Build

What is being built

A runnable FastAPI backend and React TypeScript frontend foundation.

Align

Who it supports

Later implementation agents and developers extending APIs, services, agents, tests, and pages.

Limit

What makes it useful

It proves the app can start and respond without Supabase, Qdrant, ShopAIKey, retrieval, parsing, or agents.

High-Level Architecture

The source defines a local development architecture centered on a minimal frontend, backend API, configuration layer, and health response. External systems are intentionally absent.

Local User

Runs frontend and backend locally.

React + TypeScript Frontend

Vite app shell with Axios client using VITE_API_BASE_URL.

FastAPI App

Creates app, mounts routers, configures CORS, exposes application object.

Core Configuration

Loads APP_ENV, SINGLE_USER_ID, and FRONTEND_ORIGIN.

Health Result

Returns status, service, and app_env from GET /api/health.
Storage layer, document parsing, retrieval, agents, authentication, deployment, Supabase, Qdrant, and ShopAIKey are out of scope for this plan.

Core Workflow

Create project skeleton

Backend folders, API package, core package, model reservation, tests, and frontend source files are created.

Load configuration

Backend settings read environment variables with documented defaults and frontend reads VITE_API_BASE_URL.

Start backend app

FastAPI initializes logging, CORS for the local frontend origin, and the health router under /api.

Request health endpoint

GET /api/health returns HTTP 200 with status, service, and app_env.

Verify foundation

Run backend health tests and frontend build checks before later plans depend on the layout.

Component Responsibilities

backend/app/main.py

Creates FastAPI app, includes routers, configures CORS, and exposes the app object.

backend/app/core/config.py

Defines environment-driven Settings for APP_ENV, SINGLE_USER_ID, and FRONTEND_ORIGIN.

backend/app/core/logging.py

Defines basic structured logging setup for backend startup and request debugging.

backend/app/api/health.py

Contains the APIRouter and GET /health endpoint response contract.

frontend/src/api/client.ts

Creates an Axios client configured with import.meta.env.VITE_API_BASE_URL.

frontend/src/App.tsx

Renders a minimal app shell with the app name and placeholder for future routes.

Service / Module Workflow

Settings module

Input: APP_ENV, SINGLE_USER_ID, FRONTEND_ORIGIN. Output: loaded configuration with defaults.

Logging module

Input: application startup. Output: INFO-level backend logs, including startup failure visibility.

Health API module

Input: GET request with no body. Output: status ok, service document-qa-agent, app_env.

Validation, Safety, and Quality Gates

Gate What it checks Pass condition Failure action
Backend health test GET /api/health response contract. HTTP 200 and status "ok". Fix endpoint, routing, or startup configuration.
Frontend build TypeScript and Vite build integrity. npm run build succeeds. Fix frontend compile or dependency issues.
Secret boundary No private backend keys in frontend files. Frontend .env.example contains only VITE_API_BASE_URL. Remove backend-only or private variables from frontend code.
Scope boundary No Supabase, Qdrant, ShopAIKey, auth, parsing, retrieval, or agents. Foundation remains runnable without those services. Remove out-of-scope implementation.

API, CLI, and Interface Overview

GET /api/health

Purpose: backend health check. Input: none. Output: status, service, app_env. Validation: no request validation required.

Backend test command

Command: cd backend; pytest tests/test_health.py -v. Expected: health endpoint returns HTTP 200 and status "ok".

Manual backend check

Command: uvicorn app.main:app --reload, then curl http://localhost:8000/api/health.

Frontend build check

Command: cd frontend; npm install; npm run build. Expected: TypeScript build succeeds.

Risks, Limitations, and Future Work

  • No production deployment configuration.

Future work boundaries

  • Supabase clients and storage are reserved for later plans.
  • Qdrant and ShopAIKey integration are reserved for later plans.
  • Agents and document workflows are reserved for later plans.
